# 排查：GPT 一直说「当前不是日本回合」或卡在日本回合

运行 `./run_auto.sh` 后终端会打印一行 **`[Debug]`**，例如：

```text
[Debug] currentPlayerName='Japanese' controlledPlayerName='Japan' 是否我方回合=False legal_actions条数=0
```

用这一行对照下面原因逐项排查。

---

## 1. Bridge 的 `--take` 和地图里的玩家名不一致（最常见）

- **现象**：`currentPlayerName` 和 `controlledPlayerName` 不一样（如一个是 `'Japanese'`，一个是 `'Japan'`），或 `是否我方回合=True` 但 `legal_actions条数=0`。
- **原因**：Bridge 用 `--take` 接管的玩家名必须和**地图里该玩家的名字完全一致**。默认是 `--take Japan`，而很多地图（如 Pacific 40）里日本叫 **"Japanese"**。
- **处理**：先看 Host 里当前地图「日本」显示的名字（如 Japanese / Japan），再启动 Bridge 时显式指定，例如：
  ```bash
  ./gradlew :game-app:game-bridge:run --args="--take Japanese"
  ```
  若地图里是 `Japan` 则用 `--take Japan`。改完后重启 Bridge，再跑 `./run_auto.sh`，看 Debug 里 `controlledPlayerName` 是否和 `currentPlayerName` 一致、且在我方回合时 `legal_actions条数` 是否 > 0。

---

## 2. 地图里日本名既不是 "Japanese" 也不是 "Japan"

- **现象**：Debug 里 `controlledPlayerName` 已是正确名字，但 GPT 仍说不是日本回合；或旧版 client 没有 `controlledPlayerName` 时只认 "Japanese"/"Japan"。
- **原因**：部分地图可能用 "Japan (Axis)"、"JAPAN" 等，和代码里写死的不一致。
- **处理**：用 `--take <地图里真实名字>` 启动 Bridge（见上一条）。新版本 state 里会有 `controlledPlayerName`，GPT 和 driver 都以「currentPlayerName 是否等于 controlledPlayerName」判断是否我方回合，不再依赖 "Japanese"/"Japan"。

---

## 3. 游戏还没开始

- **现象**：`territories` 为空、`connected=False`，或 Debug 里一直是「对局未开始或已断开」。
- **原因**：Host 里还没点「开始游戏」，Bridge 已连上但拿不到完整对局数据。
- **处理**：在 Host 界面**点「开始游戏」**，等对局真正开始后再看 Debug；若仍无 territories，检查 Host 和 Bridge 是否同一局、是否掉线。

---

## 4. 启动顺序或连接问题

- **现象**：Bridge 日志里没有 "Bridge connected and took player: xxx"，或 Host 里日本仍为 AI/人类。
- **原因**：先开了 Host 再开 Bridge、或 Bridge 连错 Host/端口，导致没接管到日本。
- **处理**：先启动 Host（开房、选图、选人数），再在同一台机或正确 IP/端口启动 Bridge（`--host`/`--port` 与 Host 一致）。确认 Host 里日本显示为 Bridge 控制的玩家后再跑 `./run_auto.sh`。

---

## 5. legal_actions 一直为 0 导致 GPT 误判

- **现象**：Debug 里 `是否我方回合=True` 但 `legal_actions条数=0`，GPT 仍说「当前不是日本回合」。
- **原因**：Bridge 只在「当前玩家名 == 自己接管的玩家名」时返回非空 legal_actions。若 `--take` 和地图名不一致，轮到日本时 Bridge 仍认为「不是自己回合」，于是返回 0 条，GPT 可能据此误判。
- **处理**：先按第 1 条把 `--take` 改成地图里真实名字，保证我方回合时 legal_actions 非空；同时 prompt 已改为用 `currentPlayerName === controlledPlayerName` 判断，减少误判。

---

## 6. 终端/操作上的问题

- **Bridge 没重启**：改过 `--take` 或改过代码后没重启 Bridge，仍用旧进程。
- **跑错目录**：在非 `clients/python` 下执行 `run_auto.sh`，或用了别的 Python/venv，导致用的不是当前改过的 client。
- **多开**：开了多个 Bridge 或多个 `run_auto.sh`，请求打到别的进程或旧状态。
- **端口/URL**：Bridge 不是 8081 或不在本机时，未用 `--base-url` 指定，例如：
  ```bash
  ./run_auto.sh --base-url http://127.0.0.1:8081
  ```

---

## 建议的排查顺序

1. 看**当前这一轮的 Debug 行**：记下 `currentPlayerName`、`controlledPlayerName`、`是否我方回合`、`legal_actions条数`。
2. 若 `controlledPlayerName != currentPlayerName` 或我方回合时 legal_actions=0 → 按 **第 1 条** 改 `--take` 并重启 Bridge。
3. 若 Debug 里已是「我方回合」且 legal_actions>0，但 GPT 仍说不是日本回合 → 多半是模型没按 state 判断，可再跑一轮或看 state 里是否还有 `stateError` 等异常。
4. 若一直「对局未开始或已断开」→ 按 **第 3、4 条** 确认 Host 已开始、Bridge 已连接且接管了日本。

把 Debug 那一行贴出来（或截图），可以更快精确定位问题。
